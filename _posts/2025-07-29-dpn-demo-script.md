## Introduction

회사가 Datadog 파트너를 준비하는 과정에서 Demo Certification 취득이 필요하여 아래와 같이 준비하였다.

---

## Process

1. **Datadog Partner Network(DPN) 가입**

    Datadog Demo Certification을 진행하려면 **먼저 DPN 포털에 파트너 소속 구성원으로 등록**되어 있어야 한다.

2. **Datadog Demo Certification 신청 및 교육 이수**

    DPN 등록 후, **Demo Certification 프로그램을 신청**할 수 있다.

    신청 시 제공되는 **Demo 계정 및 교육 과정을 이수**하여 시나리오 및 스크립트를 준비한다.

3. **Demo Video 제출 및 평가**

    교육을 기반으로 구성한 데모 시나리오를 **녹화하여 Demo Video 형태로 제출**한다.

    제출된 영상은 Datadog Partner Enablement 팀에서 **콘텐츠·흐름·기술 이해도 기준으로 평가**한다.

4. **Live Remote Demo 준비 및 진행**

    Demo Video 평가를 통과하면 **Live Remote Demo 세션 일정이 확정**된다.

    해당 세션에서 Datadog 담당자 앞에서 **실시간으로 데모를 수행**하며, **최종 합격 시 Demo Certification 자격을 취득한다.**

## Demo Script

### Intro

안녕하세요. OOO의 OOO 입니다.

오늘날에는 멀티 클라우드와 마이크로 서비스 아키텍처의 확산으로 인해, 클라우드 환경은 점점 더 복잡해지고 있습니다.

그만큼 장애의 원인을 빠르게 파악하고 해결하는 데 더 많은 시간과 노력이 들고 있습니다.

오늘은 이러한 복잡성을 어떻게 Datadog이 단일 플랫폼에서 효과적으로 해소할 수 있는지 보여드리도록 하겠습니다.

Datadog은 이 복잡한 데이터를 한 곳에 모으고, 그 관계를 시각화하고, 실시간으로 인사이트를 제공해서 문제 해결 시간을 대폭 줄여줍니다.

오늘 데모에서는 Datadog의 핵심이라고 할 수 있는 3개의 Pillar를 중심으로 보여드리겠습니다.

바로 메트릭(Metrics), 로그(Logs), 그리고 APM입니다.

이 세 가지 데이터를 단일 플랫폼에서 데이터를 수집하고 연결 분석함으로써, 고객의 시스템에서 일어나는 일을 End-to-End로 볼 수 있도록 돕는 것이 Datadog의 강점입니다.

### Dashboard

**Action : Dashboard 검색 후 Shop.io 대시보드로 이등**

Datadog은 수집된 데이터를 기반으로 대시보드를 자유롭게 커스터마이징 할 수 있습니다.

인프라, APM, 로그, 보안 데이터 등 여러 종류의 대시보드 템플릿을 제공하며 복잡한 설정 없이 빠르게 서비스를 시각화 할 수 있습니다.

각 위젯은 애플리케이션 모니터링, 인프라 메트릭, 로그 등 다양한 데이터를 보여줍니다.

또한, 기본 제공 메트릭 외에 내부 시스템 데이터를 Custom Metric으로 수집해 비즈니스 KPI 등도 위젯으로 구성할 수 있습니다.

개인적으로 데이터독 대시보드의 큰 장점은 바로 데이터 간 연계성이라고 생각합니다.

특정 데이터를 클릭하면, 연관된 로그나 메트릭 등 관련 데이터를 바로 확인할 수 있습니다.

예를 들어, 지금 데모에서 Web Store의 에러 데이터를 클릭해 보면 해당 시점의 로그, APM, 다른 메트릭 데이터를 같은 화면에서 연계해서 볼 수 있습니다.

**Action : Web Store Errors by Version 클릭 후 연관된 Log or trace 확인**

다음은 통합에 대해 간단하게 살펴보도록 하겠습니다.

### Integration

**Action : Integration 메뉴로 이동**

Datadog은 약 1000개 이상의 통합을 제공하며, 클릭 몇 번으로 간편하게 연동할 수 있습니다.

예를 들어, AWS의 Account ID를 입력함으로써 읽기 전용 자격 증명이 자동으로 완료되고, 클라우드 리소스 상태와 메트릭을 바로 수집하기 시작합니다.

추가로 Agent를 설치하면 서버나 애플리케이션 내부의 상세 메트릭, 로그, APM 데이터까지 더 깊이 모니터링할 수 있습니다.

### Host Map

Datadog은 여러 시스템에 대한 호스트 맵 ****기능을 통해 현재 인프라 자원의 상태를 한눈에 확인할 수 있도록 지원합니다.

**Action : Host Map 메뉴로 이동**

Datadog은 호스트, 컨테이너 등을 시각화 하여 현재 구성된 인프라 자원을 직관적으로 보여주는 호스트 맵을 제공하는데, 이 호스트 맵의 주요 특징은 태그 기반 그룹화 입니다.

태그를 활용하면 다양한 방식으로 인프라를 구분할 수 있습니다.

보시는 내용과 같이 프로덕션 / 개발 환경 별 구분을 할 수도 있고 서비스 단위나 기타 사용자 정의로 그룹화 할 수 있습니다.

또한 Datadog 호스트 맵은 Fill과 Size 기능을 제공합니다.

**Action : Fill, Size 기능 시연**

예를 들면 Fill은 인프라 자원의 CPU 사용률 등을 색상으로 시각화하고 Size는 크기를 통해 CPU 사용률을 표시합니다.

결론을 말씀드리면, Datadog 호스트 맵은 현재 사용 중인 인프라의 구성과 상태를 한눈에, 그리고 직관적으로 확인할 수 있게 해주는 강력한 기능입니다.

### Service Map

다음은 Datadog의 서비스 맵 기능입니다.

**Action : Service Map 메뉴로 이동**

서비스 맵은 현재 운영 중인 서비스 간의 연결 관계를 시각적으로 표현해 주는 기능입니다.

서비스 맵을 통해 각 서비스가 어떤 다른 서비스와 연결되어 있는지, 그리고 서비스 간 흐름이 어떻게 이루어지는지를 한눈에 파악할 수 있습니다

이 서비스는 Web-Store 서비스입니다.

**Action : Web-Store 서비스 Inspect**

이 화면에서는 Web-Store 서비스의 Upstream 및 Downstream 관계를 쉽게 확인할 수 있습니다.

문제가 있는 서비스는 색상으로 강조되어 쉽게 식별됩니다.

**Action : Web-store 서비스 View Service Information 클릭**

보시면 서비스와 연관된 Trace, Log, Metrics, 관련 호스트 정보까지 모두 통합하여 확인할 수 있습니다.

즉, 연결된 트레이스, 메트릭등을 확인하기 위해 여러가지 다른 툴들을 사용하지 않고 특정 서비스에서 문제가 발생했을때 Datadog에서 몇번의 클릭만으로 관련된 여러가지 정보들을 손쉽게 확인할 수 있다는 것이 Datadog의 큰 장점입니다.

### APM

**Action : Service Overview Data 옆 Service Page 클릭**

여기에서는 해당 서비스에 대한 APM 요청 수, 에러율, 그리고 다양한 메트릭들을 종합적으로 볼 수 있습니다.

이 화면은 서비스의 상태를 종합적으로 진단하는 데 최적화되어 있습니다.

**Action : Trace 섹션 클릭 후 트레이스 데이터 클릭**

또한, 이 화면에서는 **트레이스 데이터**도 바로 확인할 수 있습니다.

트레이스를 클릭하면 Flame Graph가 나타나고, 각 기능이나 메서드의 실행 시간, 그리고 어디에서 병목 현상이 발생했는지를 직관적으로 확인할 수 있습니다.

Flame Graph를 통해 성능 저하의 원인을 빠르게 찾아낼 수 있고, 문제가 되는 서비스나 기능을 중심으로 최적화 포인트를 쉽게 식별할 수 있습니다.

여기에 더해, 트레이스와 연결된 로그 데이터까지 함께 볼 수 있기 때문에, 어디서 데이터가 잘못 처리되었는지도 한 화면에서 추적할 수 있습니다.

### Log Management

다음은 Log Management 기능입니다.

**Action : Log Exploeer 접근 후 env:prod 등 필터 적용**

먼저, Log Explorer에 접속한 후 env:prod와 같은 태그 기반 필터를 적용할 수 있습니다.

Datadog은 클라우드 리소스의 메타 데이터와 태그를 자동으로 인식하여, 복잡한 쿼리 없이도 직관적인 필터링과 로그 탐색이 가능합니다.

**Action : Log Exploreer 필터 창의 patterns 클릭**

상단 필터 영역에서는  patterns를 통해 유사 로그 그룹을 패턴화 하여 볼 수 있습니다. 이를 통해 로그의 공통 패턴을 쉽게 식별할 수 있습니다.

또한 Add Exclusion Filter 기능을 통해 불필요한 로그를 수집하지 않도록 설정하여 분석 범위를 좁힐 수 있습니다.

**Action : Live Tail 탭으로 이동**

현재 보시는 화면은 Datadog의 Live Tail입니다.

이 기능은 수집되는 로그를 실시간으로 스트리밍하여 보여줍니다.

이 기능은 로그가 저장되거나 인덱싱되기 전 단계에서 바로 확인할 수 있어, 디버깅이나 실시간 이슈 대응에 매우 효과적입니다.

배포 직후 발생하는 이벤트나 에러 로그를 즉시 확인하고 조치할 수 있는 빠른 피드백 루프를 제공합니다.

### Monitor

**Action : Monitor 화면으로 이동**

Datadog의 Monitor는 단순한 경고 알림 기능을 넘어서, 다양한 데이터 소스에 기반한 조건 기반 경보(Alert) 를 설정할 수 있습니다.

예를 들어, CPU 사용률이 일정 수치를 넘었을 때, 혹은 특정 로그 패턴이 감지되었을 때, 사용자 지연 시간이 평소보다 높을 때 등 다양한 조건을 기준으로 자동으로 감지하고 알림을 발생시킬 수 있습니다.

대시보드, 메트릭, 로그, APM, 서비스 맵, 그리고 지금 보신 모니터링 & 알림 기능까지, Datadog은 하나의 플랫폼에서 다양한 데이터를 통합하고, 이를 바탕으로 문제를 더 빠르게 파악하고, 팀 간 협업을 원활하게 만들며, 고객 경험을 개선할 수 있도록 도와주는 도구입니다.
